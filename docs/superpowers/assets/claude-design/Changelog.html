5v5.1.0 April 18, 2026 latest

Podman, properly. Plus a faster engine swap.

First-class Podman support across every surface, a rebuilt engine selector that remembers your last pick per-project, and a Trivy pane that now shows fix versions inline. We also quietly deleted 600 lines of legacy IPC code — you should feel it.

First-class Podman, everywhere

Since v4, Podman support has been a thin compatibility layer — the renderer spoke Docker, and we translated underneath. v5.1 removes the translation. Containers, images, volumes, networks, the terminal, the stats pane — all of it now speaks libpod directly when you're on a Podman engine.

You'll notice three practical differences:

  • Rootless mounts actually show the mapped path, not the host one.
  • Quadlet .container and .network files appear inline in the Compose pane.
  • Stats are pulled from the libpod events stream — no more 2-second polling lag.
Migrating from the Docker-compat shim? Run dockerman migrate --engine=podman once — it rewrites your saved connections and re-indexes Quadlet files in the background.

Trivy scan now shows what to do next

CVE lists are only useful if you know how to act on them. The Image pane pulls the fixed-in version from the Trivy database and shows it inline, with a one-click upgrade & rebuild action that rewrites your Dockerfile and triggers a build.

New shortcuts & palette recency

The command palette (⌘;) now remembers what you used this session and floats those actions to the top. For everything else, raw muscle memory still wins:

⌘,        // open Settings
⌘E        // switch engine (docker · podman · ssh host)
⌘R        // restart focused container
⌘⇧L       // open logs in a new pane
⌘K  then "cve"   // jump to the Trivy scan pane
Breaking change on Linux The legacy IPC bridge is gone. If you ran Dockerman against a remote-only Podman socket over the old bridge, update to ssh:// connections — we published a migration guide.

Everything else

  • Image upgrade detection polls every 30 min with one-click pull & recreate.
  • Cloudflared tunnel health pings are now a live sparkline.
  • Kubernetes ConfigMap diff viewer (side-by-side or unified).
  • Stats pane refresh bumped from 2s → 1.5s.
  • Dark theme contrast raised across Events and Logs to meet WCAG AA.
  • Fixed: SSH heartbeat leaking goroutines on flapping connections. Terminal cursor on Wayland blur. WSL2 detection on Windows 11 24H2. NO_PROXY respected by the Trivy pane. extends: with relative paths in Compose. Big drag-uploads no longer freezing the UI. K8s Event timestamps now update live.
  • Removed: the legacy IPC bridge (600 lines, all of it sad) and the docker-machine detector (unused since v4.0).
"Startup is 40% faster on a cold boot; the renderer shed 22MB of resident memory on my 60-container machine." — internal benchmark, M2 Pro / macOS 15.4
5v5.0.4 April 02, 2026 security

Patch week, no drama.

A quick security rollup plus a handful of fixes that crept in after v5.0.0. If you're on v5, update.

Contains security fixes CVE-2026-1401 and CVE-2026-1588 are patched in the bundled curl and git. Updating is recommended for everyone on v5.0.x.

5v5.0.0 March 11, 2026 major

A full rewrite of the engine layer. And a command palette.

Five months in the works. The engine abstraction that used to live in the renderer moved to a sandboxed worker process with typed IPC — startup is 2.3× faster, memory is down ~40% on busy machines, and every future engine (Podman, containerd, BuildKit-only) now plugs in without a renderer refactor.

Why we rewrote it

v4's engine layer was a single giant singleton that lived in the renderer, called Docker's socket directly, and held every container's state in memory. It worked fine for a laptop with 10 containers. It did not work fine for a homelab with 80.

v5 splits the engine into a sandboxed Rust worker with a typed IPC surface. The renderer subscribes to the slice of state it cares about — the container you're looking at, the logs you asked for — and nothing else. On a 60-container machine, idle memory dropped from 380MB to 230MB.

Upgrading from v4? Your connection bookmarks, compose projects, and license all migrate automatically on first launch. If you've scripted against ~/.config/dockerman/state.json, that path is now state/engine.db (SQLite).

What's new

  • Command palette (⌘;) — every action, every object, every setting.
  • Kubernetes as a first-class module (clusters, pods, deployments, services, configmaps).
  • Cloudflared tunnel management with one-click public URLs.
  • Multi-host bookmarks with a titlebar switcher.
  • Trivy CVE scanning inside the Image pane.
  • Image registry credential manager (Docker Hub, GHCR, GCR, ECR, private).
  • Inline Compose & docker run parser with validation.
  • New Events pane with filters, search, and replay.

Removed

  • Legacy v3 "quick actions" bar — the command palette replaces it.
  • Built-in Portainer-style dashboard. It never matched the real app's data model.
  • The old blue-only theme. You can set accent: blue in Settings if you miss it.
4v4.8.0 February 14, 2026

4v4.0.0 November 19, 2025 major

The native rewrite. Goodbye, Electron.

v4 ships as native Tauri on every platform. The whole app is ~38MB down from 180MB. Memory at idle: 84MB instead of 420MB. If you remember v3 feeling slow, v4 will feel unrecognizable.

The numbers that matter

  1. Install size: 180MB → 38MB. Most of that is the dropped Chromium runtime.
  2. Idle memory: 420MB → 84MB on an M2 Mac with 20 containers.
  3. Cold start: 1.8s → 0.6s to first render; 2.4s → 1.1s to interactive.
Your v3 license works. No re-purchase, no re-activation. Just install v4 and it picks up your existing license key.
