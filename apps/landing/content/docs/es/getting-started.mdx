Usa esta página para instalar Dockerman y hacer tu primera conexión. Si estás en Windows y quieres evitar Docker Desktop, usa la ruta WSL2 integrada en la guía de la plataforma Windows: [Windows](/es/docs/platform/windows).

## Requisitos previos

<Callout type="info">
  Un daemon de Docker debe estar disponible antes de que puedas gestionar contenedores. En Windows, Dockerman puede aprovisionar uno para ti a través de la ruta integrada del Docker Engine en WSL2 en la [guía de la plataforma Windows](/es/docs/platform/windows).
</Callout>

## Instalación

<Tabs items={['macOS', 'Windows', 'Linux']}>
  <Tab>
    <Steps>
      <Step>
        ### Descarga el DMG

        Descarga el DMG de macOS desde la página de Descargas.
      </Step>
      <Step>
        ### Abre y arrastra a Aplicaciones

        Abre el DMG, luego arrastra Dockerman a Aplicaciones.
      </Step>
      <Step>
        ### Inicia Dockerman

        Inicia Dockerman desde Aplicaciones o Spotlight.
      </Step>
    </Steps>
  </Tab>
  <Tab>
    <Steps>
      <Step>
        ### Descarga el MSI

        Descarga el MSI de Windows desde la página de Descargas.
      </Step>
      <Step>
        ### Ejecuta el instalador

        Ejecuta el instalador y sigue las instrucciones.
      </Step>
      <Step>
        ### Inicia Dockerman

        Inicia Dockerman, luego usa el Asistente de Configuración WSL2 en el primer inicio si quieres la ruta integrada del Docker Engine. Consulta la [guía de la plataforma Windows](/es/docs/platform/windows).
      </Step>
    </Steps>
  </Tab>
  <Tab>
    <Steps>
      <Step>
        ### Instala el paquete

        Instala el paquete que coincida con tu distribución. Usa el build `.deb`, `.rpm`, o AppImage, luego sigue la [guía de la plataforma Linux](/es/docs/platform/linux).
      </Step>
      <Step>
        ### Añade tu usuario al grupo docker

        Ejecuta:

        ```bash
        sudo usermod -aG docker $USER
        ```

        Cierra sesión y vuelve a iniciarla para que la nueva membresía del grupo surta efecto.
      </Step>
      <Step>
        ### Inicia Dockerman

        Inicia Dockerman desde el menú de escritorio o ejecutando `dockerman`.
      </Step>
    </Steps>
  </Tab>
</Tabs>

## Requisitos del sistema

| Plataforma | Versión mínima | Arquitectura |
| --- | --- | --- |
| macOS | 11 (Big Sur) | Intel, Apple Silicon, Universal |
| Windows | 10 (1903) / 11 | x86_64 |
| Linux | kernel 5.x | x86_64, aarch64 |

Un daemon de Docker (Engine 20.10+) debe estar disponible antes de que empieces a gestionar contenedores; en Windows, Dockerman puede aprovisionar uno a través de la ruta opcional de WSL2. El soporte de Kubernetes usa el paquete opcional de k3d.
