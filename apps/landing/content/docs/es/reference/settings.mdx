Referencia de cada página de configuración. Cada sección enlaza de vuelta a la función que controla.

## Diseño del diálogo de Workspace

Configuración se abre en un diálogo tipo workspace: una barra lateral fija a la izquierda lista cada panel y el lado derecho renderiza un panel a la vez como una superficie plana desplazable (sin pestañas anidadas). El pie de licencia activada se sitúa en el chrome del diálogo, así que permanece visible independientemente del panel en el que estés. Abre Configuración desde el icono de engranaje, la bandeja del sistema o [Spotlight](/es/docs/advanced/spotlight) (`Cmd/Ctrl+;` y luego busca "Configuración").

## Controles de visibilidad de la barra lateral

Dos ajustes te permiten adaptar la barra lateral principal de la aplicación y la barra lateral de Kubernetes a tu flujo:

- **Barra lateral principal — mostrar/ocultar por elemento** — oculta entradas que nunca usas (por ejemplo *Redes* si solo usas red de host, o *Plantillas* si escribes tus propios archivos Compose). Los elementos ocultos siguen apareciendo en [Spotlight](/es/docs/advanced/spotlight).
- **Barra lateral de Kubernetes — estado de expansión por defecto por grupo** — elige qué grupos (Cargas de trabajo, Redes, Configuración y Almacenamiento, RBAC, Clúster) se abren por defecto al conectarte a un clúster. El ajuste está desacoplado del colapso en tiempo de ejecución, así que cerrar una sección durante una sesión no sobreescribe tu predeterminado.
- **Barra lateral de Kubernetes — visibilidad por elemento** — el mismo control de mostrar/ocultar que la barra lateral principal, limitado al árbol de K8s.

Restablece los valores predeterminados en cualquier momento desde el mismo panel.

## Configuración de runtime

Usa el selector de preferencia de runtime para elegir entre Auto, Docker y Podman. Auto intenta Docker primero, luego Podman. El runtime detectado se muestra debajo del selector después de una conexión exitosa. Cuando se conecta a Podman, las funciones de Compose se desactivan automáticamente.

## Configuración de Docker

Usa la configuración de Docker para elegir el tipo de conexión, establecer una ruta de socket e inspeccionar el estado de la conexión. Para conexiones basadas en SSH, consulta [Hosts remotos](/es/docs/advanced/remote-hosts).

## Panel de configuración del daemon

Usa el panel de configuración del daemon para editar `daemon.json` ya sea en una vista de formulario o en una vista JSON sin procesar. Los mirrors de registro son especialmente útiles en Windows con el motor WSL2, y la [plataforma Windows](/es/docs/platform/windows#edita-daemonjson) muestra dónde se encuentra ese archivo.

## Configuración de DNS de Kubernetes

Usa la configuración de DNS de Kubernetes para habilitar el DNS automático para los reenvíos de puertos. Consulta [Reenvío de puertos y DNS](/es/docs/kubernetes/port-forward#dns-automático) para conocer el comportamiento que controla.

## Configuración de reenvío de puertos de Kubernetes

Usa la configuración de reenvío de puertos de Kubernetes para establecer el rango de puertos locales predeterminado, el comportamiento de reintentos y la reconexión automática.

## Configuración de licencia

Usa la configuración de licencia para activar, ver y restablecer tu licencia de Dockerman. Consulta [Licencias](/es/docs/reference/licensing) para ver el flujo completo.

## Configuración de notificaciones

Usa la configuración de notificaciones para controlar qué eventos activan una notificación y si Dockerman muestra un banner en la aplicación, una notificación del sistema, o ambos.

### Canales de notificación

Gestiona canales de notificación externos (Telegram, ntfy, Gotify, Discord, Webhook) desde la página dedicada de [Notificaciones](/es/docs/homelab/notifications). Cada canal usa una URL compatible con Shoutrrr, y las credenciales se almacenan en tu Keychain del sistema.

### Reglas de alerta

Configura reglas de alerta que conectan eventos de Docker y métricas de recursos a tus canales de notificación. Consulta [Reglas de alerta](/es/docs/homelab/alerts) para la configuración completa.

### Monitoreo de actualizaciones de imágenes

Configura el monitoreo de imágenes en segundo plano y los canales de notificación por defecto desde la página de [Monitoreo de actualizaciones de imágenes](/es/docs/homelab/image-watches).

## Configuración de privacidad

Usa la configuración de privacidad para activar o desactivar el análisis del producto y los informes de errores. Ambos son opt-in por defecto.

## Configuración de atajos

Usa la configuración de atajos para personalizar o desvincular cualquier elemento listado en [Atajos de teclado](/es/docs/reference/keyboard-shortcuts).

## Configuración del terminal

Usa la configuración del terminal para cambiar la familia de fuentes, el tamaño de fuente, el estilo del cursor, el tema y la preferencia de shell predeterminada para nuevos terminales.

## Configuración de la bandeja

Usa la configuración de la bandeja para mostrar u ocultar el icono de la bandeja, minimizar Dockerman a la bandeja al cerrar, y editar las acciones del menú de la bandeja.

## Configuración de Trivy

Usa la configuración de Trivy para establecer la ruta del binario de Trivy, el intervalo de actualización de la base de datos de vulnerabilidades y los filtros de severidad predeterminados. Consulta [Escanear con Trivy](/es/docs/docker/images#escanear-con-trivy) para ver el flujo de escaneo.

## Configuración de Tareas

Gestiona el [programador de tareas en segundo plano](/es/docs/homelab/scheduler) desde este panel: habilita/deshabilita tareas individuales, cambia su programación y dispara una ejecución inmediata. Las tareas integradas (`stats_aggregation`, `stats_retention_prune`, `image_upgrade_check`) están bloqueadas contra eliminación, pero siguen siendo totalmente configurables.

## Idioma y región

Dockerman incluye localización completa para English, 简体中文, 日本語 y Español. Elige un idioma para cambiar cada cadena de la UI, incluyendo el texto de los diálogos, las alertas y la interfaz del paquete de diagnóstico. El formato de tiempo relativo sigue tu locale seleccionado a través de `Intl.RelativeTimeFormat`.
