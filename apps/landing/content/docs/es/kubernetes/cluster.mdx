Inicia un clúster local con k3d o importa un kubeconfig existente, luego gestiona el clúster desde la misma sección de Kubernetes.

## Inicia un clúster local con k3d

Usa k3d cuando quieras un clúster local desechable para desarrollo y pruebas.

<Steps>
  <Step>
    ### Abre la sección de Kubernetes

    Si no hay ningún clúster configurado, Dockerman muestra un botón de Iniciar un clúster.
  </Step>
  <Step>
    ### Haz clic en Iniciar un clúster

    Dockerman descarga el binario de k3d en el primer uso.
  </Step>
  <Step>
    ### Espera a que el clúster se levante

    El clúster local suele iniciarse en unos 10 segundos en una máquina moderna.
  </Step>
  <Step>
    ### Comienza a usar Kubernetes

    El selector de namespaces se completa y las listas de cargas de trabajo se vuelven disponibles.
  </Step>
</Steps>

<Callout type="warn">
  Dockerman descarga el binario de k3d en el primer uso. Permite HTTPS saliente o instala el binario tú mismo antes de comenzar.
</Callout>

## Importa un clúster existente

Importa un kubeconfig cuando ya tienes un clúster y quieres que Dockerman lo gestione directamente.

<Steps>
  <Step>
    ### Haz clic en Importar clúster

    Abre el flujo de importación desde la página de Clúster.
  </Step>
  <Step>
    ### Selecciona un archivo kubeconfig

    Dockerman analiza el archivo y **detecta automáticamente cada contexto** que contiene. Elige el contexto que quieras importar; Dockerman precompleta el nombre del clúster, la URL del servidor y el binding de usuario por ti.
  </Step>
  <Step>
    ### Verifica la conexión

    Dockerman sondea el servidor de API antes de guardar el clúster.
  </Step>
</Steps>

## Cambia de clúster

Usa el conmutador de clúster en la parte superior de la sección de Kubernetes para moverte entre clústeres importados y cualquier clúster local de k3d.

- La selección de clúster es **independiente del host de Docker activo**. Cambiar a otro contexto de K8s no afecta tu conexión de Docker, y viceversa.
- **Tu página actual de Kubernetes se conserva** al cambiar entre clústeres en los que existe el mismo recurso; el selector de namespace se sincroniza con el nuevo clúster.
- Tras reiniciar la aplicación, Dockerman **se reconecta al último clúster** y restaura páginas profundas como el editor YAML o la instancia de un CRD que estabas viendo.

## Elimina un clúster

Usa la acción que coincida con el tipo de clúster.

- Eliminar de Dockerman para olvidar un kubeconfig importado mientras dejas el clúster remoto en ejecución.
- Desmantelar el clúster local de k3d para eliminar el clúster y sus datos.

<Callout type="warn">
  Desmantelar un clúster de k3d es irreversible. Pierdes todas las cargas de trabajo y volúmenes persistentes creados dentro de él.
</Callout>

## Versión y estado del clúster

La página de Clúster muestra la versión del servidor, el estado actual y la lista de nodos, incluyendo las versiones de kubelet y kube-proxy de cada nodo, para que puedas detectar desviaciones de versión o una conexión rota de un vistazo.

## Nodos, PersistentVolumes y Namespaces

Cada uno de estos tiene una página dedicada en la aplicación dentro de la sección de Kubernetes. Accede a ellas desde la barra lateral de K8s:

- **Nodos** — lista de nodos con condiciones, capacidad, recursos asignables y etiquetas. Abre un nodo para ver las versiones de kubelet/kube-proxy, el runtime de contenedores y los pods programados en él.
- **PersistentVolumes** — lista de PV con ámbito de clúster, con fase, vinculación de claim, política de reclamación y capacidad. Consulta [Configuración y Almacenamiento → PersistentVolumes](/es/docs/kubernetes/config-storage#persistentvolumes) para la referencia completa.
- **Namespaces** — crea, elimina e inspecciona namespaces directamente. El selector de namespace de la barra lateral se sincroniza con esta página.

## Eventos del clúster

Abre el navegador dedicado de [Eventos](/es/docs/kubernetes/events) para filtrar por namespace, tipo (Normal/Warning), razón y objeto involucrado. Útil para triar problemas sin tener que limitarte primero a un recurso específico.
