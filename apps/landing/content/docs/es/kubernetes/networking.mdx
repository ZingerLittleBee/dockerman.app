---

Usa la página de Redes para inspeccionar cómo se mueve el tráfico a través del clúster. Lee Services, Ingresses, Endpoints y NetworkPolicies sin volver a `kubectl`.

## Services

Usa Services para ver la puerta de entrada estable para una carga de trabajo. La lista muestra el tipo de servicio, ClusterIP y puertos.

- `ClusterIP` para tráfico interno dentro del clúster.
- `NodePort` cuando necesitas un puerto a nivel de nodo para acceso rápido.
- `LoadBalancer` cuando el clúster expone un balanceador de carga externo.
- `ExternalName` cuando el Service apunta a un nombre DNS en lugar de un selector.

Los Services con un selector también muestran un botón Reenviar que inicia un [reenvío de puertos](/es/docs/kubernetes/port-forward) hacia un pod de respaldo listo.

La vista de detalle muestra la especificación completa del Service y el YAML.

## Ingresses

Usa Ingresses para inspeccionar el enrutamiento HTTP hacia el clúster. La lista muestra hosts, rutas y Services de backend, mientras que la vista de detalle muestra secretos TLS, anotaciones y la tabla de reglas.

## Endpoints

Usa Endpoints cuando necesites las direcciones de backend sin procesar detrás de un Service. Esta vista es más útil cuando un Service no tiene backends saludables.

## NetworkPolicies

Usa NetworkPolicies para entender qué pods pueden comunicarse entre sí.

<Callout type="info">
  La creación es solo en YAML — usa el editor YAML para crear nuevas NetworkPolicies.
</Callout>

## Eliminar

La eliminación en línea está disponible para Services, Ingresses y NetworkPolicies desde cada vista de lista. Consulta [Cargas de trabajo → Acciones de eliminación](/es/docs/kubernetes/workloads#acciones-de-eliminación) para ver la lista completa de recursos con eliminación en línea.
