title: "Configuración y Almacenamiento"

---

Usa Configuración y Almacenamiento para inspeccionar las fuentes de datos y los almacenes de respaldo de los que dependen las cargas de trabajo.

## ConfigMaps

Abre un ConfigMap para revisar sus claves, valores y YAML. Edita el objeto cuando necesites cambiar la configuración de una aplicación. Dockerman verifica la versión del recurso antes de guardar, de modo que recibes una advertencia de conflicto si alguien más editó el mismo ConfigMap mientras trabajabas en él. Elimina un ConfigMap desde la lista cuando ya no sea necesario.

<Callout type="info">
  Editar un ConfigMap no reinicia los pods que lo montan. Actualiza el Deployment o StatefulSet propietario si necesitas que los nuevos datos surtan efecto.
</Callout>

## Secrets

Abre un Secret para inspeccionar su tipo, claves y valores. La lista muestra tipos comunes como `Opaque`, `kubernetes.io/tls` y `kubernetes.io/dockerconfigjson`. La misma detección de conflictos por versión de recurso que en ConfigMaps se aplica al editar.

Los valores binarios de secretos se pueden descargar a un archivo local. Elimina un Secret desde la lista cuando ya no sea necesario.

<Callout type="warn">
  Revelar muestra los valores del secreto en texto plano. Cualquiera que pueda ver la pantalla en ese momento puede leerlos.
</Callout>

## PersistentVolumeClaims

Usa PersistentVolumeClaims para ver la fase, el PersistentVolume vinculado, la clase de almacenamiento, los modos de acceso y la capacidad de cada reclamación. Elimina un PVC desde la lista cuando la reclamación esté liberada y ya no sea necesaria.

## PersistentVolumes

Los PersistentVolumes tienen su propia página dedicada además de la vista por reclamación anterior. Úsala cuando necesites revisar el almacenamiento con ámbito de clúster de forma independiente de cualquier namespace.

| Columna | Descripción |
| --- | --- |
| Nombre | El nombre del PV |
| Capacidad | Capacidad total |
| Modos de acceso | `RWO`, `ROX`, `RWX` o `RWOP` |
| Política de reclamación | `Retain`, `Delete` o `Recycle` |
| Estado | `Available`, `Bound`, `Released` o `Failed` |
| Claim | El PVC `namespace/name` vinculado, si existe |
| StorageClass | La clase que provisionó el volumen |

Abre un PV para ver su especificación completa, incluyendo la fuente de volumen subyacente (driver CSI, hostPath, NFS, etc.).

## StorageClasses

Usa StorageClasses para inspeccionar las opciones de almacenamiento del clúster, incluyendo el provisionador, la política de reclamación, el modo de vinculación de volúmenes y el marcador de clase predeterminada.

## Namespaces

La página de Namespaces lista cada namespace con su fase y antigüedad, te permite crear un nuevo namespace en línea y eliminar uno (con la advertencia de cascada que devuelve la API). El selector de namespace de la barra lateral se sincroniza con esta página, por lo que las actualizaciones aquí se reflejan globalmente.
