每个设置页的参考。每个部分都链接回其控制的功能。

## 工作区对话框布局

设置以工作区风格的对话框打开：左侧固定的侧边栏列出所有面板，右侧一次渲染一个面板，作为可滚动的扁平表面（无嵌套标签）。已激活许可证的页脚位于对话框的外壳上，因此无论你在哪个面板都保持可见。可从齿轮图标、系统托盘或 [Spotlight](/zh/docs/advanced/spotlight)（`Cmd/Ctrl+;` 后搜索 "Settings"）打开设置。

## 侧边栏可见性控制

两项设置让你按工作流定制主应用侧边栏和 Kubernetes 侧边栏：

- **主侧边栏 — 单项显示/隐藏** — 隐藏你从不使用的条目（例如，仅使用主机网络时隐藏 *Networks*，或者你自己编写 Compose 文件时隐藏 *Templates*）。隐藏的条目仍会出现在 [Spotlight](/zh/docs/advanced/spotlight) 中。
- **Kubernetes 侧边栏 — 每组默认展开状态** — 选择连接到集群时哪些分组（Workloads、Networking、Config & Storage、RBAC、Cluster）默认展开。该设置与运行时折叠状态解耦，因此会话中关闭某个分组不会覆盖你的默认设置。
- **Kubernetes 侧边栏 — 每项可见性** — 与主侧边栏相同的显示/隐藏控件，作用范围限定在 K8s 树。

可在同一面板随时重置为默认。

## 运行时设置

使用运行时偏好选择器在 Auto、Docker 和 Podman 之间进行选择。Auto 模式优先尝试 Docker，然后是 Podman。成功连接后，选择器下方会显示检测到的运行时。当连接到 Podman 时，Compose 功能会自动禁用。

## Docker 设置

使用 Docker 设置来选择连接类型、设置 socket 路径并检查连接状态。对于基于 SSH 的连接，请参阅[远程主机](/zh/docs/advanced/remote-hosts)。

## Daemon 配置面板

使用 Daemon 配置面板以表单视图或原始 JSON 视图编辑 `daemon.json`。Registry mirrors 在 Windows 上使用 WSL2 Engine 时特别有用，[Windows 平台](/zh/docs/platform/windows#编辑-daemonjson)页面展示了该文件的位置。

## Kubernetes DNS 设置

使用 Kubernetes DNS 设置为端口转发启用自动 DNS。请参阅[端口转发与 DNS](/zh/docs/kubernetes/port-forward#自动-dns)了解其控制的行为。

## Kubernetes 端口转发设置

使用 Kubernetes 端口转发设置来配置默认本地端口范围、重试行为和自动重连。

## 许可证设置

使用许可证设置来激活、查看和重置你的 Dockerman 许可证。完整流程请参阅[许可证](/zh/docs/reference/licensing)。

## 通知设置

使用通知设置来控制哪些事件触发通知，以及 Dockerman 显示应用内横幅、系统通知还是两者都显示。

### 通知渠道

从专门的[通知](/zh/docs/homelab/notifications)页面管理外部通知渠道（Telegram、ntfy、Gotify、Discord、Webhook）。每个渠道使用 Shoutrrr 兼容的 URL，凭证存储在你的系统 Keychain 中。

### 告警规则

配置将 Docker 事件和资源指标连接到你的通知渠道的告警规则。完整设置请参阅[告警规则](/zh/docs/homelab/alerts)。

### 镜像升级监控

从[镜像升级监控](/zh/docs/homelab/image-watches)页面配置后台镜像监控和默认通知渠道。

## 隐私设置

使用隐私设置来开启或关闭产品分析和崩溃报告。两者默认均为可选加入。

## 快捷键设置

使用快捷键设置来自定义或解绑[键盘快捷键](/zh/docs/reference/keyboard-shortcuts)中列出的任何内容。

## 终端设置

使用终端设置来更改字体族、字体大小、光标样式、主题以及新终端的默认 shell 偏好。

## 托盘设置

使用托盘设置来显示或隐藏托盘图标、关闭时将 Dockerman 最小化到托盘，以及编辑托盘菜单操作。

## Trivy 设置

使用 Trivy 设置来配置 Trivy 二进制文件路径、漏洞数据库更新间隔和默认严重性过滤器。扫描流程请参阅[使用 Trivy 扫描](/zh/docs/docker/images#使用-trivy-扫描)。

## 任务设置

从此面板管理[后台任务调度器](/zh/docs/homelab/scheduler)：启用/禁用单个任务、更改其调度并触发立即运行。内置任务（`stats_aggregation`、`stats_retention_prune`、`image_upgrade_check`）禁止删除，但仍可完整配置。

## 语言和区域

Dockerman 提供英语、简体中文、日本語和 Español 的完整本地化。选择语言会更改每一处 UI 字符串，包括对话框文案、告警和诊断包界面。相对时间格式通过 `Intl.RelativeTimeFormat` 跟随你选择的区域设置。
