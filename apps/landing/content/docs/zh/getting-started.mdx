使用本页面安装 Dockerman 并进行首次连接。如果你在 Windows 上且希望避免使用 Docker Desktop，请使用 Windows 平台指南中的内置 WSL2 路径：[Windows](/zh/docs/platform/windows)。

## 前提条件

<Callout type="info">
  在管理容器之前，必须有一个可用的 Docker daemon。在 Windows 上，Dockerman 可以通过 [Windows 平台指南](/zh/docs/platform/windows) 中的内置 WSL2 Docker Engine 路径为你配置一个。
</Callout>

## 安装

<Tabs items={['macOS', 'Windows', 'Linux']}>
  <Tab>
    <Steps>
      <Step>
        ### 下载 DMG

        从下载页面下载 macOS DMG。
      </Step>
      <Step>
        ### 打开并拖入应用程序

        打开 DMG，然后将 Dockerman 拖入应用程序文件夹。
      </Step>
      <Step>
        ### 启动 Dockerman

        从应用程序文件夹或 Spotlight 启动 Dockerman。
      </Step>
    </Steps>
  </Tab>
  <Tab>
    <Steps>
      <Step>
        ### 下载 MSI

        从下载页面下载 Windows MSI。
      </Step>
      <Step>
        ### 运行安装程序

        运行安装程序并按照提示操作。
      </Step>
      <Step>
        ### 启动 Dockerman

        启动 Dockerman，如果需要内置 Docker Engine 路径，可在首次启动时使用 WSL2 设置向导。详见 [Windows 平台指南](/zh/docs/platform/windows)。
      </Step>
    </Steps>
  </Tab>
  <Tab>
    <Steps>
      <Step>
        ### 安装软件包

        安装适合你发行版的软件包。使用 `.deb`、`.rpm` 或 AppImage 版本，然后按照 [Linux 平台指南](/zh/docs/platform/linux) 操作。
      </Step>
      <Step>
        ### 将用户添加到 docker 组

        运行：

        ```bash
        sudo usermod -aG docker $USER
        ```

        注销并重新登录，以使新的组成员身份生效。
      </Step>
      <Step>
        ### 启动 Dockerman

        从桌面菜单启动 Dockerman，或通过运行 `dockerman` 启动。
      </Step>
    </Steps>
  </Tab>
</Tabs>

## 系统要求

| 平台 | 最低版本 | 架构 |
| --- | --- | --- |
| macOS | 11 (Big Sur) | Intel, Apple Silicon, Universal |
| Windows | 10 (1903) / 11 | x86_64 |
| Linux | kernel 5.x | x86_64, aarch64 |

在开始管理容器之前，必须有一个可用的 Docker daemon（Engine 20.10+）；在 Windows 上，Dockerman 可以通过可选的 WSL2 路径为你配置一个。Kubernetes 支持使用可选的 k3d 捆绑包。
