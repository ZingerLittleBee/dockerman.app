---
title: "远程主机 (SSH)"
description: "通过 SSH 隧道管理远程 Docker 主机，并在主机选择器中切换。"
---

Dockerman 通过 SSH 将远程 Docker 守护进程的 Unix 套接字转发到本地来连接远程主机。配置完成后，所有功能（容器、镜像、日志、终端）都与在本地主机上运行完全一致。

## 添加远程主机

只需添加一次主机，然后像切换其他 Docker 目标一样切换到它。

<Steps>
  <Step>
    ### 打开主机管理

    从托盘菜单或主机视图中打开主机管理。
  </Step>
  <Step>
    ### 点击添加主机

    点击**添加主机**。对话框会要求输入连接详情。
  </Step>
  <Step>
    ### 填写 SSH 详情

    输入主机地址、端口（默认 `22`）、用户名和认证方式。
  </Step>
  <Step>
    ### 测试并保存

    测试连接，然后保存。Dockerman 会打开测试隧道并执行 Docker ping。
  </Step>
</Steps>

## 密钥认证与密码认证

选择与你远程主机配置相匹配的 SSH 认证方式。

<Tabs items={['SSH 密钥', '密码']}>
  <Tab>
    直接粘贴私钥内容，或指定私钥文件路径给 Dockerman。

    <Callout type="warn">
      私钥在磁盘上必须是 600 权限；OpenSSH 会拒绝权限过于宽松的密钥。
    </Callout>
  </Tab>
  <Tab>
    密码认证信息存储在本地。仅建议在临时设置或共享笔记本场景中使用。
  </Tab>
</Tabs>

## 切换主机

使用顶部栏的主机切换器在 `localhost` 和所有已配置的远程主机之间切换。切换主机会重新加载新选中目标的所有数据。

## 自动重连

Dockerman 会监控 SSH 隧道健康状态，并在短暂故障后自动重连。当主机离线超过 30 秒时，会显示警告横幅。

## SSH 隧道工作原理

Dockerman 建立 SSH 会话，并将本地 Unix 套接字转发到远程的 `/var/run/docker.sock`，或你配置的套接字路径。所有 Docker API 调用都通过该隧道传输，因此数据不会离开 SSH 通道。

## 安全注意事项

<Callout type="warn">
  仅在受信任的网络上配置你信任的远程主机。SSH 凭据保存在本地机器上，因此请将存储它们的系统视为敏感设备。
</Callout>
