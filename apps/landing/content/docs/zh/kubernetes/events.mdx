---
title: 事件
description: 浏览集群事件，按命名空间、类型、原因和涉及对象进行筛选。
---

事件浏览器是用于排查 Kubernetes 事件的专属页面，无需先定位到具体资源。

## 页面内容

事件列表显示连接的集群当前 API server 暴露的所有事件，按最后出现时间排序。每行包含：

| 列 | 描述 |
| --- | --- |
| Last seen | 距最近一次出现的相对时间 |
| Type | `Normal` 或 `Warning` |
| Reason | 简短的原因码，例如 `FailedScheduling`、`BackOff`、`Pulled` |
| Object | 涉及的对象（kind/name） — 点击可跳转到其详情页 |
| Source | 发出该事件的组件或节点 |
| Count | 该事件触发的次数 |
| Message | 可读的解释 |

## 筛选

使用顶部的筛选栏缩小列表范围：

- **命名空间** — 限定到单个命名空间或查看整个集群。
- **类型** — 排查时仅显示 `Warning` 事件，或显示 `Normal` 来确认操作成功。
- **原因** — 从当前数据中存在的原因动态列表中选择。
- **对象** — 按 kind 或名称搜索，聚焦到某一工作负载。

筛选状态仅在该页面内生效；切换页面或集群会重置。

## 实时更新

事件会随 API server 的发出实时流入。新行出现在顶部，已有行的 `Count` 和 `Last seen` 会原地更新，因此你可以在重现问题时保持页面打开。

## 详情面板

点击某一行可打开详情面板，查看完整的事件载荷 — 当你需要源 `Pod` 的 UID、控制器子对象路径或原始的 `firstTimestamp`/`lastTimestamp` 值时非常有用。

## 提示

- 部署后立即筛选到 `Warning` 和受影响的命名空间，可以快速捕获调度、镜像拉取或卷挂载失败。
- 集群概览页面也会展示最近的事件；当你需要在整个集群中进行历史搜索时，使用本页面。
- API server 会按自身的策略清理事件。如果你看不到预期的事件，说明集群已经将其丢弃。
