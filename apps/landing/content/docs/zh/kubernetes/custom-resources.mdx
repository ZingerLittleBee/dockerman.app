v4.8.0 新增。使用此页面发现 CustomResourceDefinitions，浏览其实例，并以 YAML 格式编辑。

## 列出 CRD

打开 CRD 列表查看所有已安装的定义，及其组、版本、作用域和实例数量。

## 浏览资源实例

选择 CRD 以浏览其跨命名空间或集群的实例，具体取决于资源作用域。详情表格在可用时使用 CRD 的打印列。

## 查看和编辑 YAML

需要更改时，在 YAML 编辑器中打开任意实例。Dockerman 使用 CRD 模式进行验证和自动补全。

<Callout type="info">
  YAML 编辑器不限于自定义资源。在任何 Kubernetes 资源（Deployment、Service、ConfigMap 等）上打开编辑 YAML 即可查看和应用更改。Dockerman 会剥离服务器管理的字段，支持 dry-run 验证，并在保存前检测冲突。
</Callout>

## 限制

<Callout type="info">
  第一阶段仅支持自定义资源的 YAML 编辑。字段级表单计划在第二阶段提供。
</Callout>
