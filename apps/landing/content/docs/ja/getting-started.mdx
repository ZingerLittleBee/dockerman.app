このページではDockermanをインストールし、最初の接続を行います。Windowsをご使用でDocker Desktopを避けたい場合は、Windowsプラットフォームガイドの組み込みWSL2パスをご利用ください：[Windows](/ja/docs/platform/windows)。

## 前提条件

<Callout type="info">
  コンテナを管理するには、DockerまたはPodmanランタイムが事前に利用可能である必要があります。Dockermanは起動時にランタイムを自動検出します。Windowsでは、[Windowsプラットフォームガイド](/ja/docs/platform/windows)の組み込みWSL2 Docker Engineパスを通じて、DockermanがDocker daemonをプロビジョニングできます。
</Callout>

## インストール

<Tabs items={['macOS', 'Windows', 'Linux']}>
  <Tab>
    <Steps>
      <Step>
        ### DMGのダウンロード

        ダウンロードページからmacOS用DMGをダウンロードしてください。
      </Step>
      <Step>
        ### 開いてApplicationsにドラッグ

        DMGを開き、DockermanをApplicationsフォルダにドラッグしてください。
      </Step>
      <Step>
        ### Dockermanの起動

        ApplicationsまたはSpotlightからDockermanを起動してください。
      </Step>
    </Steps>
  </Tab>
  <Tab>
    <Steps>
      <Step>
        ### MSIのダウンロード

        ダウンロードページからWindows用MSIをダウンロードしてください。
      </Step>
      <Step>
        ### インストーラーの実行

        インストーラーを実行し、プロンプトに従ってください。
      </Step>
      <Step>
        ### Dockermanの起動

        Dockermanを起動し、初回起動時に組み込みDocker Engineパスをご希望の場合はWSL2セットアップウィザードをご利用ください。詳細は[Windowsプラットフォームガイド](/ja/docs/platform/windows)をご覧ください。
      </Step>
    </Steps>
  </Tab>
  <Tab>
    <Steps>
      <Step>
        ### パッケージのインストール

        お使いのディストリビューションに合ったパッケージをインストールしてください。`.deb`、`.rpm`、またはAppImageビルドをご利用いただけます。その後、[Linuxプラットフォームガイド](/ja/docs/platform/linux)に従ってください。
      </Step>
      <Step>
        ### dockerグループへのユーザー追加

        以下を実行してください：

        ```bash
        sudo usermod -aG docker $USER
        ```

        新しいグループメンバーシップが有効になるよう、ログアウトして再度ログインしてください。
      </Step>
      <Step>
        ### Dockermanの起動

        デスクトップメニューから、または`dockerman`を実行してDockermanを起動してください。
      </Step>
    </Steps>
  </Tab>
</Tabs>

## システム要件

| プラットフォーム | 最低バージョン | アーキテクチャ |
| --- | --- | --- |
| macOS | 11 (Big Sur) | Intel、Apple Silicon、Universal |
| Windows | 10 (1903) / 11 | x86_64 |
| Linux | kernel 5.x | x86_64、aarch64 |

コンテナの管理を開始する前に、Docker Engine 20.10+またはPodman 4.0+が利用可能である必要があります。Windowsでは、オプションのWSL2パスを通じてDockermanがDocker daemonをプロビジョニングできます。Kubernetesサポートにはオプションのk3dバンドルを使用します。
