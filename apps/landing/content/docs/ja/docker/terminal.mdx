---
title: "ターミナル"
description: "内蔵 xterm.js ターミナルで実行中の任意のコンテナにインタラクティブシェルで接続。"
---

xterm.jsを通じて、実行中の任意のコンテナにインタラクティブシェルを開くにはこのページを使用します。

## ターミナルを開く

コンテナリストからコンテナを選択し、ターミナルを開きます。Dockermanはコンテナ内にインタラクティブシェルがある新しいターミナルビューを開きます。

## シェルを選択する

Dockermanは`bash`、次に`sh`、次に`zsh`を試行します。別のコマンドが必要な場合は、ターミナル設定コントロールからセッションのシェルを上書きできます。

<Callout type="info">
  Alpineやdistrolessなどの最小限のイメージにはbashが含まれていない場合があります。Dockermanは自動的に`sh`にフォールバックします。
</Callout>

## コピー、ペースト、スクロール

| ショートカット | アクション |
| --- | --- |
| Ctrl/Cmd + Shift + C | 選択範囲をコピー |
| Ctrl/Cmd + Shift + V | ペースト |
| マウスホイール | スクロール |
| Ctrl/Cmd + Shift + K | ターミナルをクリア |

## テーマとフォント

組み込みのテーマから1つを選択し、[ターミナル設定](/ja/docs/reference/settings#ターミナル設定)でフォントファミリーとサイズをカスタマイズします。

## 制限事項

<Callout type="warn">
  - コンテナを停止するとターミナルセッションが終了します。
  - 長時間アイドル状態の接続はDockerによって切断される場合があるため、ターミナルを再度開いて再接続してください。
  - 一部のTUIアプリを起動する前に`TERM=xterm-256color`が必要な場合があります。
</Callout>
