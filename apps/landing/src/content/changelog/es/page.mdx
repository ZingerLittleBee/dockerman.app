## Kubernetes Multi-Cluster, Vigilancia de Imágenes y Bases del Homelab

Esta versión amplía la gestión de Kubernetes con cambio multi-cluster, dry-run de YAML y páginas dedicadas para nodos, PV y namespaces; añade búsqueda de logs, un servicio de vigilancia de actualizaciones de imagen con notificaciones, backup/restore de bind mounts y la primera fase del roadmap Homelab; además aterriza un workspace de configuración, cobertura completa de i18n y un patrón estandarizado de tablas de datos.

### ✨ Funcionalidades

- ☸️ <Bold>Mejoras de Kubernetes:</Bold> superficie de gestión de cluster ampliada
  - 🔀 Cambio multi-cluster: cambia entre clusters de Kubernetes de forma independiente de los hosts Docker
  - 🧪 Vista previa YAML dry-run: compara cambios antes de aplicar mediante dry-run del servidor
  - 📣 Explorador de eventos: página de eventos del cluster con filtros por endpoint
  - 🖥️ Páginas Node / PV / Namespace: páginas dedicadas de gestión para nodos, volúmenes persistentes y namespaces
  - 🔍 Descubrimiento de contextos kubeconfig: detección automática de contextos en el diálogo de importación
  - 🛡️ Manejo tipado de 403: `K8sListResult<T>` expone errores de permisos en las vistas de lista
  - 🗑️ Botones de eliminar: añadidos para Services, Ingresses, CronJobs, DaemonSets, Deployments, StatefulSets y NetworkPolicies
- 🔍 <Bold>Búsqueda de logs:</Bold> busca en logs con resaltado de palabras clave, regex, alternar mayúsculas/minúsculas y navegación entre coincidencias
- 🎛️ <Bold>Controles de log unificados:</Bold> presets de rango temporal, selector de tail y filtros de consulta consolidados en una sola toolbar
- 📈 <Bold>Gráfico histórico de stats:</Bold> historial de CPU/memoria con selector de rango temporal de 7 niveles
- 📝 <Bold>Editor YAML de Compose:</Bold> editor basado en Monaco con comandos Tauri en backend para editar y aplicar archivos Compose
- 🔄 <Bold>Conversor Docker Run → Compose:</Bold> página y entradas de menú dedicadas que traducen comandos `docker run` a YAML Compose
- 🔔 <Bold>Vigilancia de actualizaciones de imagen:</Bold> servicio de suscripción en segundo plano para detectar actualizaciones
  - 📬 Página de suscripción y UI de ajustes con controles por canal
  - 🔐 Resolutor de credenciales del registro con manejo fail-closed ante ambigüedad
  - 🕒 Planificador con seguimiento de cursor de historial y notificaciones por lotes
  - 🔗 Manejador de deep links `dockerman://`
- 💾 <Bold>Backup y restore de bind mounts:</Bold> incluye los datos de bind mount soportados en los backups y los remapea o restaura por mount al restaurar
- 🏡 <Bold>Roadmap Homelab (Fase 1–3 parcial):</Bold> base más una selección de elementos no MVP
  - 🧰 Bundle de diagnóstico: empaquetado en un clic de logs, inspect y estado del host para soporte
  - 🚨 Reglas de alerta preestablecidas: reglas integradas de restart-loop y crash de contenedor con guarda de borrado y restablecer a defaults
  - 📝 Editor visual de .env: página `/env-editor?path=...` con edición clave/valor, conservación de comentarios y líneas en blanco, y Save & Apply atómico
  - 📊 Comparativa de stats multi-contenedor: gráficos de CPU/memoria lado a lado para hasta 6 contenedores con selecciones persistentes
  - 📂 Reforzamiento del editor de archivos del contenedor: guardado preservando modo, tope de edición de 1 MB y detección de binario por byte NUL
- 🌐 <Bold>Cobertura completa de i18n:</Bold> reemplazadas las cadenas hardcoded restantes por llamadas i18n y traducciones para español, japonés y chino
- ⚙️ <Bold>Diálogo workspace de Ajustes:</Bold> ajustes movidos a un diálogo estilo workspace con paneles aplanados, barra lateral refinada y pie de licencia activada simplificado
- 👁️ <Bold>Controles de visibilidad de la barra lateral:</Bold> mostrar/ocultar por elemento en la barra principal y, para Kubernetes, controles de estado de expansión por defecto y visibilidad por grupo
- 🧱 <Bold>Patrón estandarizado de tabla de datos:</Bold> base sobre TanStack Table v8 con redimensionado/orden de columnas, anchos persistentes, búsqueda global, tooltips por truncado y estado vacío centrado en viewport — aplicado a K8s `event`, `pod`, `deployment`, `daemon-set`, `stateful-set`, `job`, `cron-job`, `service`, `ingress`, `config-map`, `pv`, `namespace` y `port-forward`

- 🚨 <Bold>Metadatos de alertas recientes:</Bold> contenedor, hora local y nombre de regla mostrados en la lista de alertas con una fila accesible para reglas preestablecidas

## Soporte del Runtime Podman

Esta versión introduce soporte de primera clase para el runtime Podman junto con Docker, con detección automática de runtime, selector de preferencia por host y un sistema extensible de capabilities para el gating de funciones.

### ✨ Funcionalidades

- 🐙 <Bold>Soporte Podman:</Bold> soporte de primera clase para el runtime Podman junto con Docker
  - 🔍 Detección de runtime: detección automática de Docker, Podman o runtimes compatibles vía API
  - ⚙️ Preferencia de runtime: selector por host y global (Auto / Docker / Podman)
  - 🔌 Resolución de socket: descubrimiento automático de sockets Podman rootless/rootful en Linux y macOS
  - 🌐 Probing SSH remoto: resolución de socket consciente del runtime para hosts SSH remotos
  - 🏷️ Badge de runtime: el switcher de host en la barra lateral muestra el runtime detectado con badge a color
  - 📊 Integración con el dashboard: el tipo de runtime se muestra en la tarjeta de información del sistema
  - 🔧 Feedback de conexión: runtime detectado mostrado en la salida del terminal al conectar
  - 🚫 Gate de Compose: plantillas de stack y acciones Compose ocultas automáticamente al conectar a Podman
  - 🔎 Conciencia en Spotlight: la paleta de comandos omite datos de Compose cuando se conecta a Podman
  - 🧩 Capa de capabilities: sistema extensible de capabilities de runtime para gating futuro

### 🔧 Mejoras

- 🔄 <Bold>Cambio de runtime:</Bold> la página de plantillas se actualiza dinámicamente al alternar entre hosts Docker y Podman
- 🛡️ <Bold>Aislamiento de Spotlight:</Bold> los chequeos de runtime de Spotlight usan comandos Tauri directos sin efectos en el store

Esta versión principal introduce gestión completa de Kubernetes, túneles Cloudflared para exposición a internet con un clic, detección automática de actualizaciones de imágenes con rollback, un panel de información de contenedores renovado y dos nuevos idiomas.
